# 游戏设计文档 (GDD) v2 — 代号：三国新生

> **核心理念**：三国志11的大地图回合制战棋骨骼 + 信长之野望16新生的内政/知行血肉  
> **引擎**：Godot 4.x  
> **开发模式**：全 AI 辅助编程  
> **地图**：四角格（菱形等距网格），交错排列，对标三11原版尺寸  
> **版本**：MVP v0.1

---

## 一、游戏概览

### 1.1 一句话描述

在中国巨型等距地图上，以回合制指挥部队征战；每座城池的领地在地图上可见，知行武将代管郡县、建设城下町——一切都在一张地图上发生。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **地图即一切** | 战斗、内政、发展、建设全部发生在一张大地图上，无切屏 |
| **所见即所得** | 领地发展、城下町、郡界都直接在地图上可视 |
| **武将即灵魂** | 每个武将有自己的意志（呈报），不是玩家的木偶 |
| **知行分离** | 玩家是君主，武将代管郡县，形成"委托-监督"循环 |
| **回合制** | 每个回合 = 一旬（10天），一年 36 回合 |
| **数据即地图** | 地图全部由 JSON 数据文件定义，可直接编辑 |
| **无单挑/舌战** | 不切屏到小游戏，战斗全部在大地图上结算 |

### 1.3 与参考作品的差异

| 系统 | 三国志11 | 信野16新生 | 本作（三国新生） |
|------|---------|-----------|----------------|
| 地图 | 等距网格（四角格）大地图 | 郡/国 3D 地图 | **等距网格（继承三11）** |
| 战斗 | 大地图回合制 | 实时会战 | **大地图回合制（继承三11）** |
| 内政 | 城数值面板 | 知行/郡/呈报 | **地图可见郡域 + 知行 + 呈报（继承新生）** |
| 武将 | 五维+特技 | 能力+特性+呈报 | **五维+特技+呈报** |
| 设施 | 地图设施建造 | 城下町 | **地图设施 + 城下町（继承两者）** |
| 领地 | 无 | 郡制 | **地图上郡界可视的知行制** |
| 单挑/舌战 | 有 | 无 | **无（继承新生）** |

## 二、地图系统（核心）

### 2.1 网格规格

```
网格类型：     四角格（菱形/等距网格 Isometric Diamond Grid）
排列方式：     交错型（Staggered）— 奇偶行偏移半格
视觉风格：     类似三11，大地图俯瞰视角

顶点规格：
  - 宽度（X）：200 顶点
  - 高度（Y）：180 顶点
  - 总顶点数：36,000 个可站立位置
  - 有效顶点（不在山脉/深水）：约 25,000+
  - 一个顶点 ≈ 实际约 15-20 公里

坐标系统：
  顶点坐标用 (x, y) 表示，x ∈ [0, 199], y ∈ [0, 179]
  
  屏幕坐标换算（菱形网格）：
    screen_x = (x - y) * TILE_HALF_W + MAP_OFFSET_X
    screen_y = (x + y) * TILE_HALF_H + MAP_OFFSET_Y - height * HEIGHT_STEP
  
  相邻顶点（4方向 + 4对角）：
    (x-1, y), (x+1, y), (x, y-1), (x, y+1)        ← 主方向
    (x-1, y-1), (x+1, y-1), (x-1, y+1), (x+1, y+1) ← 对角

高程系统：
  每个顶点有一个高度值 (0-10)，影响移动消耗和攻防加成
  低 → 高移动消耗 +1 per level；高 → 低不影响
```

### 2.2 数据驱动的可编辑地图

**核心设计：地图不是硬编码的，而是从 JSON 数据文件加载。** 你可以直接编辑 JSON 来调整任何地图元素的位置。

```
地图数据文件结构：

data/map/
├── terrain.json       ← 每个顶点的地形类型和高程
├── rivers.json        ← 河流路径（折线顶点列表）
├── cities.json        ← 城市位置、名称、下辖郡范围
├── passes.json        ← 关隘位置、名称、所属城市
├── harbors.json       ← 港口位置、名称
├── roads.json         ← 主要道路（顶点列表）
└── resources.json     ← 资源点（矿山/农田/村落）位置
```

**terrain.json 格式示例：**
```json
{
  "width": 200,
  "height": 180,
  "vertices": [
    {"x": 0, "y": 0, "terrain": "deep_water", "height": 0},
    {"x": 1, "y": 0, "terrain": "deep_water", "height": 0},
    {"x": 0, "y": 1, "terrain": "plain",      "height": 2},
    ...
  ]
}
```

**rivers.json 格式示例：**
```json
{
  "rivers": [
    {
      "name": "黄河",
      "path": [[45,20], [46,21], [47,21], [48,22], ...]
    },
    {
      "name": "长江",
      "path": [[30,100], [31,100], [32,101], ...]
    }
  ]
}
```

**cities.json 格式示例：**
```json
{
  "cities": [
    {
      "id": "luoyang",
      "name": "洛阳",
      "position": {"x": 98, "y": 65},
      "max_durability": 3000,
      "counties": [
        {"id": "luoyang_01", "name": "河南", "center": {"x": 99, "y": 66}, "tiles": [[98,65],[99,65],[99,66],...]},
        {"id": "luoyang_02", "name": "河内", "center": {"x": 97, "y": 64}, "tiles": [...]}
      ]
    }
  ]
}
```

### 2.3 地图编辑流程

玩家（你）调整地图的方法：

```
方法一：直接编辑 JSON
  1. 打开 data/map/cities.json
  2. 修改城市 position 坐标
  3. 重进游戏，城市位置改变

方法二：未来内置地图编辑器（Phase 6+）
  1. 在游戏中按 F1 进入编辑模式
  2. 点击拖拽城市/关隘
  3. 画笔调整地形
  4. 保存 → 更新 JSON 文件
```

### 2.4 地形类型

| 地形 | 移动消耗 | 防御加成 | 可建设 | 特效 |
|------|---------|---------|--------|------|
| 平原 | 1 | 0% | ✅ | - |
| 草地 | 1 | +5% | ✅ | - |
| 森林 | 2 | +15% | ❌ | 伏兵可能 |
| 密林 | 3 | +25% | ❌ | 视野-2 |
| 丘陵 | 2 | +10% | ✅ | - |
| 山地 | 3 | +30% | ❌ | 落石计可发动 |
| 高山 | 不可通行 | - | ❌ | 完全阻挡 |
| 湿地 | 2 | -10% | ❌ | 火计伤害+50% |
| 浅滩 | 2 | -5% | ❌ | 可渡河 |
| 河流（深水） | 不可通行 | - | ❌ | 只有水军可通过 |
| 道路 | 0.5 | 0% | ✅ | 城市间的主干道 |
| 关口 | 1 | +40% | ✅ | 特殊防御建筑 |
| 港口 | 1 | +20% | ✅ | 可编成水军 |

### 2.5 地图要素

**城市（治所）**：
- 占据一个顶点位置
- 控制周围一定范围的郡域
- 有耐久值（石垣耐久），归零则城市陷落
- MVP：15 城，完整版：50+ 城
- 城市列表：襄平 / 北平 / 邺 / 晋阳 / 平原 / 北海 / 洛阳 / 长安 / 许昌 / 陈留 / 濮阳 / 下邳 / 寿春 / 建业 / 襄阳（可通过 JSON 任意增减）

**关隘**：
- 占据一个顶点，横跨关键道路
- 类似城市有耐久值，但不可发展内政
- 可驻军防守
- MVP 关隘：虎牢关 / 函谷关 / 剑阁 / 阳平关 / 葭萌关 / 武关 / 壶关 / 汜水关
- **所有关隘的位置和属性可在 JSON 中调整**

**港口**：
- 占据一个顶点，位于河边/海边
- 可编成水军
- 可驻军

**河流**：
- 由折线定义（JSON 中的顶点序列）
- 深水格不可通行（除非水军）
- 浅滩格可徒步渡河
- 桥梁位置由道路数据定义
- **你可以通过编辑 rivers.json 调整河流走向**

**道路（主干道）**：
- 连接主要城市和关隘
- 移动消耗 0.5（快速行军）
- 由顶点序列定义
- **可编辑，用于调整地图平衡**

### 2.6 郡域可视系统

```
每个城市下辖 2-6 个郡（county）
郡的边界在地图上以半透明颜色显示：

  - 自己的郡域 → 势力色 + 20% 透明
  - 邻国的郡域 → 红色警告带
  - 中立的郡域 → 灰色

视觉层次：
  Level 1：地形底图（草/树/山/水）
  Level 2：郡域颜色覆盖（势力色半透明）
  Level 3：城下町建筑（小图标贴在顶点上）
  Level 4：部队/NPC（单位 sprite）
  Level 5：网格线（可开关）
  
郡的开发度影响地图表现：
  - 农业开发度 0-300 → 零星农田 sprite
  - 农业开发度 300-700 → 中等农田
  - 农业开发度 700-1000 → 密集农田 + 风车
  - 商业开发度影响市场摊位密集度
  - 兵舍开发度影响练兵场规模
```

## 三、武将系统

### 3.1 武将属性

```
基础五维（1-100）：
  统率：影响部队防御力、带兵上限
  武力：影响攻击力
  智力：影响计略成功率
  政治：影响内政开发效率
  魅力：影响登用成功率、外交效果、人口增长

衍生属性：
  带兵上限 = 统率 × 80 + 官职加成
  俸禄 = f(五维总和, 官职)
  
  忠诚度 (0-100)：对所属势力的忠诚
  野心 (0-100)：影响独立/谋反的概率阈值
  义理 (0-100)：抵抗流言/策反的坚毅度
  
  寿命：出生年 - 死亡年（到年自动判定死亡概率）

相性（0-255 环形数值线）：
  曹操系 ~50  /  刘备系 ~120  /  孙坚系 ~200
  武将间相性差越小，配合越好，登用越容易
  差值 > 100 则可能"厌恶"
```

### 3.2 特技系统

每个武将 0-3 个特技：

```
军事系：
  飞将   — 无视地形限制，移动如飞
  神将   — 统率高于对方时，普通攻击必暴击
  勇将   — 武力高于对方时，普通攻击必暴击
  洞察   — 免疫所有计略效果
  藤甲   — 物理防御 ×2，但火计伤害 ×3
  强行   — 部队移动力 +5
  疾走   — 击破一支敌军后，本回合可再移动一次
  斗神   — 枪兵战法必暴击
  骑神   — 骑兵战法必暴击
  弓神   — 弩兵战法必暴击
  水神   — 水上战法必暴击 + 水上移动力翻倍
  攻城   — 对城耐久伤害 ×1.5
  守城   — 守城时城耐久损耗 -50%
  屯田   — 部队驻扎时粮食消耗 -50%

内政系：
  能吏   — 内政开发效率 ×2（农业/商业/兵舍）
  富豪   — 所在城金钱收入 +50%
  米道   — 所在城粮食收入 +50%
  徵税   — 每回合额外获得所在城金钱 5%
  徵兵   — 征兵获得兵力 +30%
  发明   — 科技研究速度 +50%
  筑城   — 城耐久恢复速度 ×2
  法治   — 所在郡治安自然恢复 +50%

计略系：
  神算   — 智力高于对方时，计略必成功
  连环   — 计略命中后，相邻敌军单位也被影响
  鬼谋   — 计略施放射程 +2
  火攻   — 火计伤害 ×1.5，着火范围 +1
  百出   — 计略士气消耗 -50%
  看破   — 敌方对自己施计成功率 -30%

外交系：
  论客   — 外交交涉成功率 +30%
  倾国   — 对异性武将登用/流言成功率 ×2
  仁政   — 同城武将忠诚自然衰减 -50%
  威压   — 邻接势力对你的宣战欲望降低
```

### 3.3 人际关系

```
关系类型：
  义兄弟（最多2人）：同部队战力加成，永远不背叛
  配偶：互相忠诚度联动，一荣俱荣
  亲爱：同部队战力微增，登用容易
  厌恶：无法同部队，登用该武将则厌恶者忠诚下降
  血缘：父子/兄弟，忠诚不易下降
```

### 3.4 身份与官职

```
身份层级：
  俘虏 → 在野 → 未发现 → 一般武将 → 知郡事 → 太守 → 都督 → 军师 → 君主

官职体系（汉官，依势力地位开放）：
  无官 → 校尉 → 中郎将 → 裨将军 → 偏将军 → 
  前/后/左/右将军 → 车骑将军 → 骠骑将军 → 大将军
  
官职效果：
  + 带兵上限
  + 知郡数量上限
  + 外交/登用时的身份加成
  - 增加俸禄（维护费）
```

## 四、内政系统（地图可视化的知行制）

### 4.1 核心理念：地图即内政面板

不是打开一个内政面板看数字。**你在看地图的时候，就看得到你的领地发展状况。**

```
视觉 → 信息 → 决策：
  地图上看到某郡农田稀疏 → 点击该郡 → 看到农业开发度低 → 派遣高政治武将 → 
  几回合后回来 → 地图上该郡农田变密集了
```

### 4.2 知行/郡制度

```
城市-郡-知行的关系：

  治所（主城顶点 + 城下町区域）
  │
  ├── 郡 A（地图上一片顶点区域，有郡界）
  │   │  知行：武将 A（政治78）
  │   │  农业：620/1000  ← 地图上可见农田密度
  │   │  商业：350/1000  ← 地图上可见集市
  │   │  兵舍：200/1000  ← 地图上可见练兵场
  │   │  人口：12,000
  │   │  治安：85/100
  │   └── 城下町设施：[农田×2, 市场×1]
  │
  ├── 郡 B（另一片顶点区域）
  │   │  知行：武将 B（政治92）
  │   │  ...
  │   └── 城下町设施：[农田×1, 兵营×1, 马场×1]
  │
  └── 郡 C
      │  知行：武将 C（政治55）
      └── 城下町设施：[市场×2, 学问所×1]

郡的视觉边界：
  - 郡界 = 该郡包含的顶点集合的外轮廓
  - 用势力色半透明填充
  - 鼠标悬停在某个顶点上 → 高亮显示该顶点所属的郡
  - 点击顶点 → 显示郡信息面板

任命知行的流程：
  1. 在地图上点击一个郡
  2. 右侧面板显示该郡现状 + 可任命的武将列表
  3. 选择武将 → 确认 → 该武将变成"知郡事"
  4. 武将头像出现在该郡的中心位置
```

### 4.3 城下町建设（地图可见）

```
城下町 = 在某个郡的特定顶点上建造的设施

建造流程：
  1. 进入该郡的城下町模式（或直接在地图上操作）
  2. 该郡内的可建造顶点高亮
  3. 选择要建造的设施 → 消耗金钱/时间
  4. 建造完成后，设施 sprite 出现在该顶点上
  5. 设施影响该郡的每回合产出

设施类型：
  农田     — 该郡农业开发速度 +20/回合（地图显示：耕田 sprite）
  市场     — 该郡商业开发速度 +20/回合（地图显示：集市 sprite）
  兵营     — 该郡兵舍开发速度 +20/回合（地图显示：帐篷+练兵场）
  工房     — 该郡对科技研究的贡献 +1/回合
  学问所   — 该郡每月概率发现在野武将
  酒馆     — 该郡武将忠诚自然衰减 -50%
  马场     — 解锁该郡编制骑兵的能力（地图显示：马厩）
  弓场     — 解锁该郡编制弩兵的能力
  造船所   — 解锁该郡编制水军的能力（需在河流/海岸边）
  锻冶场   — 该郡出征部队攻击力 +5%
  粮仓     — 该郡可存储粮食上限 +50%

每郡可建设施上限：
  - 小郡（< 30 个顶点）：3 个设施
  - 中郡（30-50 个顶点）：5 个设施
  - 大郡（> 50 个顶点）：7 个设施

设施升级：
  - 每个设施有 3 级
  - 升级花费递增，效果递增
  - Lv3 农田在地图上显示为有风车的繁荣农田
```

### 4.4 呈报系统

```
每回合，各郡的知行武将概率向君主提出呈报。

呈报 = 地图上该郡冒出一个气泡图标，点击查看详情

呈报频率 = 武将政治 × 0.2 + 武将智力 × 0.1（百分比/回合）
即政治 80 的武将，每回合约 16-18% 概率呈报

呈报类型：

【开发类】「XX郡农田可扩张，请批准」
  消耗：500 金 → 结果：该郡农业开发度 +50
  地图变化：该郡农田 sprite 变密

【商业类】「XX郡商贾求设市集」
  消耗：300 金 → 结果：商业开发度 +40，获得一次性 200 金

【军事类】「XX郡捕获贼寇，可编入行伍」
  结果：该郡兵力 +300

【人事类】「XX郡发现一隐士，名 YYY」
  结果：触发登用窗口

【基建类】「XX郡水利失修，建议整备」
  消耗：800 金 → 结果：农业 +80，该郡抗灾害能力提升

【治安类】「XX郡盗贼猖獗，请派兵镇压」
  选择：亲征（武力判定）/ 派兵（金+兵力消耗）/ 放任（治安-20）

玩家可选择：
  [批准] → 消耗资源，执行呈报
  [拒绝] → 无消耗，武将忠诚可能微降
  [搁置] → 下回合可能再次呈报
```

### 4.5 技术研究

```
势力科技树，技术研究速度 = Σ(各郡工房数量) × 负责研究武将的政治 / 10

军事技术（每项 4 级）：
  枪兵术 Lv1-4  → 枪兵攻击 +5%/+10%/+15%/+20%，解锁高级战法
  骑兵术 Lv1-4  → 骑兵攻击 + 移动力
  弩兵术 Lv1-4  → 弩兵射程 + 暴击率
  水军术 Lv1-4  → 水上移动 + 战法
  攻城术 Lv1-4  → 对城伤害 + 解锁冲车/井阑/投石/木兽
  防卫术 Lv1-4  → 城耐久 + 箭楼威力

内政技术（每项 4 级）：
  农政 Lv1-4   → 粮食产量 +10%/级
  商政 Lv1-4   → 金钱收入 +10%/级
  民兵制 Lv1-4 → 征兵效率 + 预备兵上限
  发明 Lv1-4   → 研究速度 + 解锁高级城下町设施
```

## 五、军事/战斗系统

### 5.1 兵种

```
基础兵种（兵种克制：枪 → 骑 → 弩 → 枪，水军独立）：

  枪兵  — 克制骑兵（对骑伤害+30%），被弩兵克制
        移动 4 / 攻 C / 防 A / 造价低
  
  骑兵  — 克制弩兵（对弩伤害+30%），被枪兵克制
        移动 6 / 攻 A / 防 B / 造价高
  
  弩兵  — 克制枪兵（对枪伤害+30%），被骑兵克制
        移动 3 / 攻 B / 防 C / 远程2格
  
  水军  — 水上唯一兵种，在河流/海洋中正常移动
        移动 5(水中) / 攻 B / 防 B

特殊兵种（需技术 + 特定条件解锁）：
  虎豹骑   — 曹操势力专属骑兵，移 8 / 攻 S
  陷阵营   — 高顺专属枪兵，防 S
  元戎弩   — 诸葛亮专属弩兵，射程+1 / 暴击率+20%
  藤甲兵   — 南方势力，防 S 但火计伤害 ×3

攻城兵器（需攻城技术解锁）：
  冲车     — 对城耐久 ×3 伤害，移 2
  井阑     — 远程 3 格攻击城内守军，移 2
  投石车   — 远程 4 格范围伤害，移 1
  木兽     — 近距离火焰伤害，对建筑有效，移 3
```

### 5.2 部队编制

```
一支部队 = 主将 ×1 + 副将 ×0~2 + 兵种 + 兵力 + 粮草

兵力上限 = 主将带兵上限（统率 × 80 + 官职加成）
每支部队最少 1000 兵

部队属性计算：
  攻击力 = (主将武力×0.7 + 副将平均武力×0.3) × 兵种攻击系数 × 兵力系数
  防御力 = (主将统率×0.7 + 副将平均统率×0.3) × 兵种防御系数 × 兵力系数
  计略智力 = max(主将智力, 副将智力)

  兵力系数 = √(当前兵力 / 1000)，上限 3.0（即 9000 兵饱和）
  
  士气：初始 100，每回合自然 -2，战斗 -5，中计 -10~20
  士气 < 50：战力 ×0.7
  士气 < 30：战力 ×0.5，可能溃散
  士气归零 → 部队溃散（兵力返回出发城 50%）

副将价值：
  - 提供属性加权
  - 提供额外特技（主+副特技全部生效）
  - 亲爱/义兄弟 → 额外 10%/20% 战力加成
  - 厌恶 → 士气每回合额外 -5

兵粮消耗：
  每 1000 兵每回合消耗 10 粮食
  出城时携带粮食 = 出发城粮食 - 携带量
  粮尽 → 士气每回合 -20，兵力每回合 -10%
```

### 5.3 战法系统

```
战法消耗士气，成功率和武将武力/智力相关：
  基础成功率 = 60% + (发动武将相关属性 - 目标武将相关属性) × 0.5%

枪兵战法：
  Lv1 突刺     — 单体 ×1.2 伤害，击退 1 格（士气 10）
  Lv2 螺旋突刺 — 单体 ×1.4 伤害，混乱概率 30%（士气 20）
  Lv3 二段突刺 — 直线 2 格穿透伤害（士气 30）

骑兵战法：
  Lv1 突进     — 单体 ×1.3 伤害，自身穿越到敌后方 1 格（士气 10）
  Lv2 突击     — 单体 ×1.5 伤害，击退 2 格（士气 20）
  Lv3 疾风突进 — 3 格直线穿透（士气 30）

弩兵战法：
  Lv1 火矢     — 远程 ×1.1 + 着火概率 20%（士气 10）
  Lv2 贯射     — 穿透目标，对后方单位也造成伤害（士气 20）
  Lv3 乱射     — 十字 4 方向伤害（士气 30）

水军战法（需水军技术）：
  Lv1 火船     — 单体 ×1.2 + 着火（士气 10）
  Lv2 撞击     — 击退敌方 2 格（士气 20）
  Lv3 猛火船   — 前方 3 格扇形着火（士气 30）
```

### 5.4 计略系统

```
计略消耗士气，成功率 = 40% + (己方智力 - 敌方智力) × 0.6%

  火计   — 在目标格放火，持续 3 回合（士气 15）
  混乱   — 目标 2 回合内无法行动（士气 20）
  伪报   — 目标下回合向反方向移动（士气 15）
  镇静   — 解除友军异常状态（士气 5）
  落石   — 限定山地，对下方邻格敌军造成 1.5× 伤害（士气 25）
  落穴   — 在邻格挖陷阱，敌军踏上触发：伤害+混乱（士气 20）
  灭火   — 熄灭目标格的火焰（士气 5）

计略成功率上限 95%，下限 5%
特技修饰：
  神算 → 智力高于对方时 100%（被洞察免疫）
  鬼谋 → +2 施放射程
  连环 → 相邻敌单位也被影响
  洞察 → 自己免疫
```

### 5.5 攻城规则

```
攻城条件：部队移动到与城市相邻的顶点

攻城流程：
  1. 攻城方每回合对城耐久造成伤害
     伤害 = max(兵力 × 0.05, 攻城兵器伤害)
  2. 守城方可反击（守军兵力 × 守将统率 × 0.03）
  3. 城耐久归零 → 城市陷落，易主

城防：
  耐久值：1000（小城）~ 3000（大城）+ 技术加成
  守军：城内驻留部队参与防御
  箭楼：如果城下有箭楼，自动对围城敌军射击

特殊攻城：
  火攻城墙 — 智力 80+ 可放火烧城耐久
  水攻 — 特定城市（下邳/樊城等），特定条件可水淹
  劝降 — 守将忠诚低 + 我方魅力高时可劝（见外交）
```

### 5.6 地图设施（军事用途）

```
在地图上直接建造的军事设施（区别于城下町）：

  砦（阵）   — 占据 1 顶点，范围内的友军防御 +20%
  箭楼       — 占据 1 顶点，自动攻击射程内敌军
  军乐台     — 占据 1 顶点，范围内友军士气回复 +5/回合
  石兵八阵   — 占据 1 顶点，经过的敌军概率混乱（需要特技"鬼谋"）

建造条件：
  - 在我方郡域内或中立地格
  - 消耗金钱和建造回合
  - 建造部队需停留 2-3 回合
  - 设施有耐久，可被敌军破坏
```

## 六、外交系统

### 6.1 外交指令

```
基础外交：
  同盟 (12回合/4月)   — 互不攻击，可共同出兵
  停战 (6回合/2月)    — 临时停火
  劝降                — 对方归降（需满足严格条件）
  破弃盟约            — 单方面撕毁（全势力好感 -30）

合作外交：
  共同出兵  — 邀请同盟在同一回合攻击同一目标城市
  请求援军  — 请求同盟派兵到指定位置
  交换俘虏  — 用俘虏换回我方被俘武将
  婚姻联姻  — 武将通婚，同盟关系更深

谋略外交：
  流言      — 降低目标城市武将的忠诚度
  驱虎吞狼  — 挑拨势力 A 攻击势力 B
  二虎竞食  — 同时向 A 和 B 示好，让它们互相猜忌
  离间      — 离间武将和其君主的关系

使者选择：
  - 选魅力高的武将担任使者
  - 有特技"论客"则成功率 +30%
  - 使者被俘风险（敌对势力可能扣押）

成功率：
  基础 = 30% + (使者魅力 - 对方君主魅力/2) × 0.5% 
        + 相性接近加成 + 已有关系加成
        - 实力差距惩罚（弱国 vs 强国更难）
  上限 95%，下限 5%
```

### 6.2 包围网

```
触发条件（满足任一）：
  - 某势力城市数 ≥ 全图 30%
  - 某势力总兵力 ≥ 全图 35%
  - 某势力连续吞并 3 个以上势力

触发后：
  - 所有与该势力邻接的势力自动结成"反XX同盟"
  - 同盟持续 24 回合（8个月）
  - 同盟内互相共享该势力的兵力情报
  - 同盟内部不可互相攻击
  - 到期后若条件仍满足，可续盟
```

## 八、AI 势力行为

### 8.1 AI 决策框架

```
每回合 AI 势力按以下优先级决策：

1. 生存评估
   - 是否正在被攻击？→ 动员防守
   - 邻国兵力远大于我？→ 外交求和或同盟

2. 扩张评估
   - 邻接空城或弱城？→ 出兵攻取
   - 同盟邀请共同出兵？→ 评估收益
   - 包围网成员？→ 协同出兵

3. 内政评估
   - 金钱/粮食不足？→ 发展内政
   - 有未分配的知郡事位置？→ 任命高政治武将
   - 有可建造的城下町位置？→ 建造
   - 科技可升级？→ 投入研究

AI 性格（影响权重）：
  扩张型（曹操/孙策）  → 扩张权重 ×2.0
  发展型（刘表/陶谦）  → 内政权重 ×2.0
  平衡型（孙权/刘备）  → 均衡发展
  保守型（刘璋/张鲁）  → 防守优先，极少主动出兵
```

### 10.2 菱形网格渲染方案

```
Godot 中没有内置的"菱形顶点网格"TileMap。

实施方案：自定义绘制

方案 A（MVP 推荐）：TileMap + 等距布局
  - 使用 Godot TileMap 的 Isometric 模式
  - 每个 tile 是一个菱形
  - 地形、郡色、设施都用 TileMap 层
  - 优点：易实现、有编辑器支持
  - 缺点：灵活性略低

方案 B（完整版推荐）：自定义 Mesh + Shader
  - 生成一个平面 Mesh，顶点对应地图顶点
  - 用 Vertex Shader 变换实现等距投影
  - 用 Fragment Shader 混色实现郡色覆盖
  - 优点：完全灵活，视觉效果更好
  - 缺点：实现量大

MVP 采用方案 A，后续可迁移到方案 B
```

### 10.3 坐标转换工具 (GridUtils.gd)

```gdscript
# 网格坐标 → 屏幕坐标
func grid_to_screen(grid_x: int, grid_y: int, height: int = 0) -> Vector2:
    var tile_w = 64   # 菱形半宽
    var tile_h = 32   # 菱形半高
    var height_step = 8  # 每级高程偏移
    
    var sx = (grid_x - grid_y) * (tile_w / 2.0) + MAP_OFFSET_X
    var sy = (grid_x + grid_y) * (tile_h / 2.0) - height * height_step + MAP_OFFSET_Y
    return Vector2(sx, sy)

# 屏幕坐标 → 最近的网格坐标
func screen_to_grid(screen_x: float, screen_y: float) -> Vector2i:
    # 逆向变换，找到最近的顶点
    var tile_w = 64
    var tile_h = 32
    var rel_x = screen_x - MAP_OFFSET_X
    var rel_y = screen_y - MAP_OFFSET_Y
    var gx_f = rel_x / (tile_w / 2.0) + rel_y / (tile_h / 2.0)
    var gy_f = rel_y / (tile_h / 2.0) - rel_x / (tile_w / 2.0)
    return Vector2i(roundi(gx_f), roundi(gy_f))

# 网格距离（曼哈顿距离）
func grid_distance(a: Vector2i, b: Vector2i) -> int:
    return abs(a.x - b.x) + abs(a.y - b.y)
```

### B. 设计哲学

```
1. 地图是家，不是UI——你看着地图就能玩大部分内容
2. 武将不是工具人——他们有自己的意志（呈报），你管方向，他们管执行
3. 编辑权归玩家——不满意河流走向？改 JSON。不满意城市位置？拖一下。
4. 不追求复杂小游戏——不切屏单挑/舌战，战斗都在大地图上解决
5. 每次启动都能看到进展——渐进开发，Phase 0 就能看到地图
```
